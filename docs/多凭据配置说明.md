# 关于不同设备的凭据问题

## 您的问题

> "那有没有一种可能不同设备对该SSH登录凭据的权限是不一样的呢？"

## 回答

**是的！您说得完全对。** 这是现实中非常常见的情况。

## 为什么会这样？

### 常见原因：

1. **🏢 不同部门**
   - 财务部门的设备：finance_admin / Finance@2024
   - IT部门的设备：it_admin / ITAdmin@2024

2. **🔒 不同安全级别**
   - 普通办公区：admin / Office123
   - 核心机房：coreadmin / V3ryS3cur3P@ss!

3. **🛠️ 老旧设备**
   - 新设备（2024）：admin / NewPass2024
   - 老设备（2018）：oldadmin / LegacyPass2018

4. **🏭 不同厂商**
   - Cisco 设备：cisco_admin / Cisco@123
   - 华为设备：huawei_admin / Huawei@456
   - H3C 设备：h3c_admin / H3C@789

## 解决方案

### ✅ 本系统已经支持！

#### 方法1：统一凭据（最简单）

所有设备用同一个账号密码：

```bash
# .env 文件
DEVICE_USERNAME=admin
DEVICE_PASSWORD=SamePassword123
```

#### 方法2：混合凭据（推荐，灵活）

大部分设备用默认凭据，特殊设备单独配置：

**步骤1：配置默认凭据（.env）**
```bash
# 大部分设备使用这个
DEVICE_USERNAME=admin
DEVICE_PASSWORD=CommonPass123
```

**步骤2：为特殊设备配置专用凭据（devices.yaml）**
```yaml
devices:
  # 普通设备 - 自动用 .env 默认凭据
  - ip: "192.168.1.1"
    device_type: "cisco_ios"
    # 没配 username/password，自动用 .env

  - ip: "192.168.1.2"
    device_type: "cisco_ios"
    # 同上

  # 老旧设备 - 单独配置
  - ip: "192.168.2.1"
    device_type: "ios"
    username: "oldadmin"      # 这台设备用专门的账号
    password: "OldPass2018"

  # 核心设备 - 高安全凭据
  - ip: "10.0.0.1"
    device_type: "cisco_ios"
    username: "coreadmin"
    password: "V3ryS3cur3!"

  # 华为设备 - 不同账号
  - ip: "192.168.3.1"
    device_type: "huawei"
    username: "huawei_admin"
    password: "Huawei@2024"
```

## 工作原理

系统会这样查找凭据：

1. **先查** devices.yaml 中是否为该设备配置了专用凭据
2. **如果有**：使用设备专用的 username/password
3. **如果没有**：使用 .env 中的默认 DEVICE_USERNAME/DEVICE_PASSWORD

## 优势

- ✅ **简单**：大部分设备只需配置 .env
- ✅ **灵活**：特殊设备可以单独配置
- ✅ **安全**：不同安全级别用不同密码
- ✅ **实用**：适应现实环境的复杂情况

## 实际例子

假设您有 100 台设备：
- 80 台使用相同凭据 → 只配置 .env
- 10 台老设备用旧密码 → 在 devices.yaml 中单独配置这10台
- 5 台核心设备用高安全密码 → 在 devices.yaml 中单独配置这5台
- 5 台华为设备用不同账号 → 在 devices.yaml 中单独配置这5台

**结果**：
- .env：1次配置（默认凭据）
- devices.yaml：只配置 20 台特殊设备（不是全部100台）
